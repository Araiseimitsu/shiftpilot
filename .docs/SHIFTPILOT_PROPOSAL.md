# ShiftPilot 提案書

## 1. 概要

**ShiftPilot** は、当番・シフトの割当を支援するデスクトップアプリケーションである。ルールと希望（NG）を入力し、指定期間の担当表案を生成・確認・エクスポートする。

## 2. 目的と前提

- **日勤（週末）**: 各日について **2 枠（1番手・2番手）** のみを扱う。**3番手は設けない**。
- **夜勤（週単位など運用で定義する単位）**: **1 枠（1番手）のみ**を扱う。**2番手は設けない**。
- **人員プール**: 従来「週末の3番手」「夜勤の2番手」に相当する役割にのみ属していたメンバーは、**自動割当の候補に含めない**（設定で明示的に除外リストとして管理する）。
- 生成結果は運用で確定し、履歴として蓄積できること。

## 3. 機能要件（案）

### 3.1 必須

| ID | 内容 |
|----|------|
| F-01 | 指定期間のスケジュール案を自動生成する |
| F-02 | メンバー一覧・有効/無効・**割当対象外**の管理 |
| F-03 | NG（日付・期間・全体休業日など）の設定 |
| F-04 | 日勤と夜勤が**同一日に同一人物で重複しない**ことの検証 |
| F-05 | 担当間隔・公平性など、設定可能な制約の適用 |
| F-06 | 結果の表形式表示・CSV（または同等）エクスポート |
| F-07 | 履歴データの読み込み（過去担当の参照・統計に利用） |

### 3.2 あるとよい

| ID | 内容 |
|----|------|
| F-08 | 複数案（バリエーション）の生成 |
| F-09 | 制約違反・解なし時の理由表示 |
| F-10 | 設定の YAML/JSON 等でのバックアップ・差分管理 |

## 4. 制約・ビジネスルール（設定化）

以下は既定値を持ち、設定ファイルまたは画面で変更可能とする想定。

- 日勤が割り当たる**対象日**（例: 土日のみ、祝日の扱いは設定で選択）
- 夜勤の**週の始まり**と**1週間あたりの割当単位**
- 同一人物の**日勤同士の最小間隔**
- 同一人物の**夜勤同士の最小間隔**
- **夜勤直後の日勤禁止日数**（またはクールダウン）
- **公平性**（担当回数・最終担当からの経過などに基づくスコアリング）
- **全体NG日**（会社休業日など、日勤・夜勤の扱いをルールに従ってスキップまたは調整）

※ 3番手・夜勤2番手は**存在しない**ため、それら専用の間隔や index 切替ルールは**不要**。

## 5. データモデル（案）

### 5.1 履歴・出力（CSV 等）

例（列名は実装時に確定）:

- `date`: 日付
- `shift_category`: `Day` | `Night`
- `shift_index`: 日勤は `1` または `2`、夜勤は `1` のみ
- `person_name`: 氏名

### 5.2 設定

- メンバー属性（表示名、割当対象か、備考）
- NG 定義（メンバー別・全体・期間）
- 制約パラメータ（間隔、公平性、ローテ期間の切り方）

## 6. 技術スタック

| 層 | 採用 |
|----|------|
| 言語・ランタイム | Python（バージョンは `uv` 管理で固定） |
| 依存管理 | **uv** |
| デスクトップシェル | **pywebview** |
| UI | **Svelte** + **Tailwind CSS** + **shadcn-svelte**（shadcn の Svelte 向けポート） |
| API | アプリ内で HTTP サーバ（例: FastAPI）を立て、webview から呼ぶ方式、または pywebview のブリッジのみ（実装段階で選択） |
| 配布 | **PyInstaller** により Windows 向け **exe** を生成 |

### 6.1 ビルドの流れ（案）

1. フロントエンドをビルドし、静的成果物を Python パッケージのリソースとして配置できる形にする。
2. PyInstaller の spec で上記リソースとエントリポイントを同梱し、`sys._MEIPASS` 等でパス解決する。
3. 起動時にバックエンドを内部起動し、pywebview がローカル URL を表示する。

## 7. アーキテクチャ（案）

```
shiftpilot/
├── pyproject.toml          # uv / Python 依存
├── src/shiftpilot/         # コアロジック（制約・ソルバ・入出力）
├── backend/                # API 層（必要なら）
├── frontend/               # Svelte + Tailwind + shadcn
├── config/                 # サンプル設定・スキーマ
├── scripts/                # ビルド・exe 用
└── tests/
```

- **コア**: 純粋な Python（日付・候補生成・制約チェック・探索または最適化）
- **UI**: 期間指定、設定編集、生成実行、結果テーブル、エクスポート
- **結合**: デスクトップは pywebview のみをユーザーに見せる

## 8. ソルバ方針（案）

- 第一版: 制約チェック付きの**構築的アルゴリズム**（高速・実装が軽い）
- 解が得られない場合: 失敗理由を日付・枠・候補不足などに分解して表示
- 必要に応じて第二版で **制約プログラミング（CP-SAT 等）** へ拡張し、解の存在性と最適化を強化

## 9. 非機能要件

- **オフライン**: 外部 API 必須にしない（祝日マスタを同梱または任意インポートにするなど）
- **ログ**: 生成失敗・設定読込エラーをファイルまたは画面で追えること
- **識別子**: Windows 用 AppUserModelID 等、タスクバー表示に必要な属性を整理

## 10. マイルストーン（案）

| フェーズ | 内容 |
|----------|------|
| M1 | リポジトリ初期化（uv + frontend テンプレ）、設定読込、CSV 入出力 |
| M2 | コア制約とソルバ v1、単体テスト |
| M3 | Svelte UI（生成・表示・エクスポート）、pywebview 統合 |
| M4 | PyInstaller で exe 化、インストール手順書 |

## 11. オープン項目（実装前に決定）

- 日勤の「週末」の定義に**祝日カレンダー**を含めるか（手動 NG のみで足りるか）
- 夜勤を「月曜始まりの週の 1 枠」で固定するか、設定で変えるか
- 履歴 CSV の**既存ファイルとの互換**（列・列挙値）をどこまで保証するか
